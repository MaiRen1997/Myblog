Go中的结构体-(复合)数据类型

# Go中的结构体

# 关于结构体

  1. Golang中没有“类”的概念，Golang中的结构体和其他语言中的类有点相似。和其他面向对象语言中的类相比，Golang中的结构体具有更高的扩展性和灵活性。

  2. Golang中的基础数据类型可以装示一些事物的基本属性，但是当我们想表达一个事物的全部或部分属性时，这时候再用单一的基本数据类型就无法满足需求了，Golang提供了一种自定义数据类型，可以封装多个基本数据类型，这种数据类型叫结构体，英文名称struct。也就是我们可以通过struct来定义自己的类型了。

# 概述

Golang中通过type关键词定义一个结构体，需要注意的是，数组和结构体都是值类型，在这个和Java是有区别的

# 结构体方法和接收者

  1. 在go语言中，没有类的概念但是可以给类型（结构体，自定义类型）定义方法。所谓方法就是定义了接收者的函数。接收者的概念就类似于其他语言中的this 或者self。

  2. 方法的定义格式如下：

    • func (接收者变量 接收者类型) 方法名(参数列表)(返回参数) {
    函数体
}

    • 其中

      • 接收者变量：接收者中的参数变量名在命名时，官方建议使用接收者类型名的第一个小写字母，而不是self、this之类的命名。例如，Person类型的接收者变量应该命名为p，Connector类型的接收者变量应该命名为c等。、

      • 接收者类型：接收者类型和参数类似，可以是指针类型和非指针类型。

        • 非指针类型：表示不修改结构体的内容

# 嵌套结构体的字段名冲突

  1. 嵌套结构体内部可能存在相同的字段名，这个时候为了避免歧义，需要指定具体的内嵌结构体的字段。（例如，父结构体中的字段 和 子结构体中的字段相似）

  2. 默认会从父结构体中寻找，如果找不到的话，再去子结构体中在找

  3. 如果子类的结构体中，同时存在着两个相同的字段，那么这个时候就会报错了，因为程序不知道修改那个字段的为准。

# 结构体的继承

  1. 结构体的继承，其实就类似于结构体的嵌套，如下所示，我们定义了两个结构体，分别是Animal 和 Dog，其中每个结构体都有各自的方法，然后通过Dog结构体 继承于 Animal结构体

  2. 通过结构体嵌套即可完成继承

    • // 用户结构体
type Animal struct {
	name string
}
func (a Animal) run() {
	fmt.Printf("%v 在运动 \n", a.name)
}
// 子结构体
type Dog struct {
	age int
	// 通过结构体嵌套，完成继承
	Animal
}
func (dog Dog) wang()  {
	fmt.Printf("%v 在汪汪汪 \n", dog.name)
}

func main() {
	var dog = Dog{
		age: 10,
		Animal: Animal{
			name: "阿帕奇",
		},
	}
	dog.run();
	dog.wang();
}

# 分析

# 组合

  1. 明确字段名：address 是一个命名字段

  2. 访问方式：必须通过字段名访问

    user := User{}
user.address.city = "北京"  // 必须使用字段名
    1
    2

  3. 语义：表示"有一个"的关系（User 有一个 Address）

  4. 类型明确：Address 是一个独立的类型

  5. 无方法提升：Address 的方法不会自动提升到 User

# 嵌入

  1. 匿名嵌入：只有类型名，没有字段名

  2. 访问方式：可以直接访问嵌入类型的字段和方法

    dog := Dog{}
dog.name = "旺财"  // 直接访问，不需要 dog.Animal.name
dog.run()        // 方法自动提升
    1
    2
    3

  3. 语义：表示"是一个"的关系（Dog 是一个 Animal），实现类似继承的效果

  4. 方法提升：Animal 的所有方法自动提升到 Dog

  5. 可以重写：Dog 可以定义同名方法来"重写"Animal 的方法

# 关键对比

特性 组合（第一种） 嵌入（第二种）
语法 有字段名：address Address 无字段名：Animal
访问方式 必须通过字段名：user.address.city 可以直接访问：dog.name
方法提升 ❌ 没有方法提升 ✅ 方法自动提升
语义 "有一个"（has-a） "是一个"（is-a）
类型关系 组合关系 类似继承关系
覆盖/重写 不能重写嵌套结构体的方法 可以定义同名方法覆盖

# 格式

  1. Tag是结构体的元信息，可以在运行的时候通过反射的机制读取出来。Tag在结构体字段的后方定义，由一对反引号包裹起来，具体的格式如下：

    • ` key1："value1" key2："value2" `
      1

  2. 结构体tag由一个或多个键值对组成。键与值使用冒号分隔，值用双引号括起来。同一个结构体字段可以设置多个键值对tag，不同的键值对之间使用空格分隔。

  3. 注意事项：为结构体编写Tag时，必须严格遵守键值对的规则。结构体标签的解析代码的容错能力很差，一旦格式写错，编译和运行时都不会提示任何错误，通过反射也无法正确取值。例如不要在key和value之间添加空格。

  4. 如下所示，我们通过tag标签，来转换字符串的key

    • // 定义一个Student体，使用结构体标签

# 控制序列化：控制 JSON/XML/YAML 等格式的编解码行为

package main

import (
    "encoding/json"
    "fmt"
)

type Product struct {
    ID        int     `json:"id"`                    // JSON 字段名为 "id"
    Name      string  `json:"product_name"`          // 映射为不同的 JSON 字段名
    Price     float64 `json:"price,omitempty"`       // omitempty: 零值时省略
    IsActive  bool    `json:"-"`                     // -: 忽略此字段
    Secret    string  `json:"secret,omitempty"`      // 零值时会省略
    CreatedAt string  `json:"created_at,string"`     // 指定类型为字符串
}

func main() {
    p := Product{
        ID:        1,
        Name:      "Laptop",
        Price:     0,           // 零值
        IsActive:  true,
        Secret:    "",          // 空字符串
        CreatedAt: "2024-01-01",
    }
    
    // 序列化为 JSON
    data, _ := json.Marshal(p)
    fmt.Println(string(data))
    // 输出: {"id":1,"product_name":"Laptop","created_at":"\"2024-01-01\""}
    // Price 被省略（omitempty），IsActive 被忽略（-），Secret 被省略（空字符串）
}

# 数据验证：定义字段的验证规则

package main

import (
    "fmt"
    "github.com/go-playground/validator/v10"
)

type LoginForm struct {
    Username string `validate:"required,min=3,max=20"`
    Email    string `validate:"required,email"`
    Age      int    `validate:"gte=18,lte=100"`
    Password string `validate:"required,min=8"`
}

func main() {
    validate := validator.New()
    
    form := LoginForm{
        Username: "ab",      // 太短
        Email:    "invalid", // 不是有效邮箱
        Age:      16,        // 太小
        Password: "123",     // 太短
    }
    
    err := validate.Struct(form)
    if err != nil {
        for _, err := range err.(validator.ValidationErrors) {
            fmt.Printf("字段 %s 验证失败: %s\n", err.Field(), err.Tag())
        }
    }
}

package main

import (
    "fmt"
    "reflect"
)

type User struct {
    ID       int    `db:"id" gorm:"primaryKey"`
    Username string `db:"username" gorm:"type:varchar(100);not null"`
    Email    string `db:"email" gorm:"uniqueIndex"`
    Password string `db:"-"`  // 不存储到数据库
}

func main() {
    u := User{}
    t := reflect.TypeOf(u)
    
    for i := 0; i < t.NumField(); i++ {
        field := t.Field(i)
        fmt.Printf("字段: %s, db标签: %s, gorm标签: %s\n",
            field.Name,
            field.Tag.Get("db"),
            field.Tag.Get("gorm"))
    }
}

# 自定义用途：根据业务需要定义任意元信息

package main

import (
    "fmt"
    "reflect"
)

type Employee struct {
    Name    string `label:"员工姓名" length:"50" required:"true"`
    Salary  int    `label:"月薪" unit:"元" min:"3000"`
    Dept    string `label:"部门" enum:"技术部,市场部,财务部"`
}

func parseTags(obj interface{}) {
    t := reflect.TypeOf(obj).Elem()
    
    for i := 0; i < t.NumField(); i++ {
        field := t.Field(i)
        tag := field.Tag
        
        fmt.Printf("字段: %s\n", field.Name)
        fmt.Printf("  显示标签: %s\n", tag.Get("label"))
        fmt.Printf("  最大长度: %s\n", tag.Get("length"))
        fmt.Printf("  是否必填: %s\n", tag.Get("required"))
        fmt.Printf("  单位: %s\n", tag.Get("unit"))
        fmt.Printf("  最小值: %s\n", tag.Get("min"))
        fmt.Printf("  枚举值: %s\n", tag.Get("enum"))
        fmt.Println()
    }
}

func main() {
    e := Employee{}
    parseTags(&e)
}
