## 1. 项目初始化与管理

| 命令            | 说明                       | 示例                   |
| :-------------- | :------------------------- | :--------------------- |
| `uv init`       | 初始化新项目               | `uv init`              |
| `uv init --app` | 初始化应用程序项目（默认） | `uv init --app my-app` |
| `uv init --lib` | 初始化库项目               | `uv init --lib my-lib` |

------

## 2. 依赖管理

| 命令                         | 说明                      | 示例                        |
| :--------------------------- | :------------------------ | :-------------------------- |
| `uv add`                     | 添加依赖包                | `uv add requests`           |
| `uv add --dev`               | 添加开发依赖              | `uv add pytest --dev`       |
| `uv add "package>=1.0,<2.0"` | 添加指定版本的依赖        | `uv add "numpy>=1.24,<2.0"` |
| `uv remove`                  | 移除依赖包                | `uv remove requests`        |
| `uv sync`                    | 同步项目依赖（安装/更新） | `uv sync`                   |
| `uv sync --no-dev`           | 同步但不安装开发依赖      | `uv sync --no-dev`          |
| `uv lock`                    | 生成/更新锁文件           | `uv lock`                   |
| `uv pip list`                | 列出已安装的包            | `uv pip list`               |
| `uv pip show`                | 显示包的详细信息          | `uv pip show requests`      |

------

## 3. Python 版本管理

| 命令                | 说明                       | 示例                     |
| :------------------ | :------------------------- | :----------------------- |
| `uv python list`    | 列出可用的 Python 版本     | `uv python list`         |
| `uv python install` | 安装 Python 版本           | `uv python install 3.12` |
| `uv python pin`     | 固定项目使用的 Python 版本 | `uv python pin 3.12`     |
| `uv python find`    | 查找 Python 解释器位置     | `uv python find`         |

------

## 4. 虚拟环境管理

| 命令                    | 说明                     | 示例                    |
| :---------------------- | :----------------------- | :---------------------- |
| `uv venv`               | 创建虚拟环境             | `uv venv`               |
| `uv venv --python 3.12` | 指定 Python 版本创建环境 | `uv venv --python 3.12` |
| `uv venv .venv`         | 指定虚拟环境路径         | `uv venv my-env`        |

------

## 5. 运行与执行

| 命令            | 说明                          | 示例                          |
| :-------------- | :---------------------------- | :---------------------------- |
| `uv run`        | 在项目环境中运行命令          | `uv run python script.py`     |
| `uv run python` | 在项目环境中启动 Python REPL  | `uv run python`               |
| `uv run pytest` | 运行测试（如果安装了 pytest） | `uv run pytest`               |
| `uv run --with` | 临时添加包后运行              | `uv run --with black black .` |

------

## 6. 工具管理

| 命令                | 说明                     | 示例                       |
| :------------------ | :----------------------- | :------------------------- |
| `uv tool install`   | 安装全局工具             | `uv tool install ruff`     |
| `uv tool run`       | 运行全局工具（无需安装） | `uv tool run ruff check .` |
| `uv tool upgrade`   | 升级全局工具             | `uv tool upgrade ruff`     |
| `uv tool list`      | 列出已安装的全局工具     | `uv tool list`             |
| `uv tool uninstall` | 卸载全局工具             | `uv tool uninstall ruff`   |

## 最常用的工作流命令

对于日常开发，最常用的命令序列是：

```bash
# 1. 初始化项目
uv init

# 2. 添加依赖
uv add requests pandas

# 3. 添加开发依赖
uv add pytest black --dev

# 4. 运行代码
uv run main.py

# 5. 修改 Python 版本（如果需要）
uv python pin 3.12

# 6. 同步环境
uv sync

# 7. 查看依赖
uv tree
```

## 命令对比：uv vs pip + venv

| 操作         | 传统方式                    | uv 方式               |
| :----------- | :-------------------------- | :-------------------- |
| 创建虚拟环境 | `python -m venv .venv`      | `uv venv`             |
| 激活环境     | `source .venv/bin/activate` | 不需要（用 `uv run`） |
| 安装包       | `pip install requests`      | `uv add requests`     |
| 安装开发依赖 | `pip install pytest`        | `uv add pytest --dev` |
| 运行脚本     | `python script.py`          | `uv run script.py`    |
| 生成锁文件   | 手动 `pip freeze`           | 自动生成 `uv.lock`    |
| 同步环境     | 手动操作                    | `uv sync`             |

## 小技巧

1. **自动补全**：可以配置 shell 自动补全

   ```bash
   # 查看自动补全配置方法
   uv help --shell-completion bash
   ```
   
2. **查看帮助**：任何命令都可以加 `--help` 查看详细用法

   ```bash
   uv --help
   uv add --help
   ```
   
3. **使用国内镜像**：设置环境变量加速下载

   ```bash
   export UV_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple
   ```
   

这些命令覆盖了 `uv` 在日常开发中的大部分使用场景。`uv` 的设计理念是让开发者专注于代码，而不是环境管理，所以它尽可能地简化了操作流程。
