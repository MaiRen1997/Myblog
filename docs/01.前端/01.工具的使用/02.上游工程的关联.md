## 添加上游工程(第一次)

1. 添加上游项目地址

   ```bash
   git remote add upstream <UPSTREAM_URL>
   ```

2. 验证远程配置

   ```sh
   git remote -v
   ```

## 抓取上游最新代码

```bash
git fetch upstream
```

## 切换到目标分支

```bash
git checkout main
```

## 合并上游分支到本地

```
git merge upstream/main
```

## 冲突合并

如果有冲突，进行冲突合并

## 将合并结果推送到自己的fork

```bash
git push origin main
```

## 在功能分支合并代码

1. 如果你在功能分支，则先切回主分支

2.  然后，从主分支合并到功能分支。

   ```bash
   git checkout feature-branch
   git merge main
   ```

3. 然后解决冲突并推送

## 使用rebase替代方法

1. 如果你偏好线性历史，可使用rebase

2. 注意，rebase会重写提交记录

   ```bash
   git checkout main
   git fetch upstream
   git rebase upstream/main
   ```

3. 随后强制推送到远程(如需要)

   ```bash
   git push --force-with-lease origin main
   ```
